# GitHub Draft Releases

This guide explains how to use `teddy-releaser` to create a draft GitHub release
for Teddy.

When `--github-draft-release` is provided, `teddy-releaser` builds the release
artifacts as normal and then uses the GitHub CLI (`gh`) to create a draft release
in the `teddyful/teddy` repository.

The draft release uses:

- tag name: `v<x.y.z>`, for example `v0.0.15`;
- title: `v<x.y.z>`, for example `v0.0.15`;
- GitHub-generated release notes;
- uploaded `.tgz`, `.zip`, and checksum files generated by `teddy-releaser`.

## Prerequisites

You need:

- Git;
- Node.js;
- npm dependencies installed in `teddy-releaser`;
- GitHub CLI (`gh`);
- a GitHub account with permission to create releases in `teddyful/teddy`.

## Install GitHub CLI

On Ubuntu/Linux, try:

```bash
sudo apt update
sudo apt install gh
```

If your package repositories do not include `gh`, follow GitHub's official
installation instructions:

<https://github.com/cli/cli/blob/trunk/docs/install_linux.md>

Verify the installation:

```bash
gh --version
```

## Authenticate GitHub CLI

Authenticate with GitHub:

```bash
gh auth login
```

Then verify authentication:

```bash
gh auth status
```

The authenticated account must be able to create draft releases and upload
release assets to `teddyful/teddy`.

## Release Notes Configuration

GitHub-generated release notes are configured in the Teddy repository, not in
`teddy-releaser`.

The configuration file should live at:

```text
teddy/.github/release.yml
```

GitHub uses this file when `gh release create --generate-notes` is run against
`teddyful/teddy`.

Release note categories are normally based on merged pull request labels. For
best results, use labelled PRs for future releases.

## Build Artifacts and Create Draft Release

From the `teddy-releaser` repository root, run:

```bash
npm run build -- --repo /absolute/path/to/teddy --github-draft-release
```

The releaser will:

1. Validate the Teddy repository.
2. Confirm the Teddy branch is named `release-x.y.z`.
3. Confirm the branch version matches `package.json`.
4. Confirm the Teddy working tree is clean.
5. Run `npm test`.
6. Run `npm run test:upgrade`.
7. Create release archives and checksums under
   `working/releases/<version>/`.
8. Create a draft GitHub release.
9. Upload the generated release artifacts.

Expected local artifacts:

```text
working/releases/<version>/teddy-<version>.tgz
working/releases/<version>/teddy-<version>.zip
working/releases/<version>/teddy-<version>-checksums.txt
```

Expected draft GitHub release:

```text
Tag: v<version>
Title: v<version>
Draft: yes
Generated notes: yes
```

## Manual Equivalent

The draft release step is equivalent to:

```bash
gh release create v0.0.15 \
  working/releases/0.0.15/teddy-0.0.15.tgz \
  working/releases/0.0.15/teddy-0.0.15.zip \
  working/releases/0.0.15/teddy-0.0.15-checksums.txt \
  --repo teddyful/teddy \
  --draft \
  --generate-notes \
  --title "v0.0.15"
```

`teddy-releaser` constructs this command automatically from the version in
Teddy's `package.json`.

## Before Publishing the Draft

Before publishing the draft release:

- Confirm the generated release notes are accurate.
- Confirm all expected artifacts are attached.
- Confirm the tag name and title match the intended version.
- Confirm the checksums file contains both archive hashes.
- Optionally download the assets from the draft release and verify checksums.
- Optionally test an upgrade from a disposable Teddy installation.

## Troubleshooting

If `gh` is not installed, the release build will fail at the draft release stage.
Install `gh`, authenticate, then run the command again.

If authentication fails, run:

```bash
gh auth login
```

If the GitHub account lacks permission to create releases, authenticate as an
account with the correct repository permissions.

If the release already exists, delete or edit the existing draft/release on
GitHub, or create a new release version.

If generated release notes are not grouped as expected, confirm that
`teddy/.github/release.yml` exists and that merged PRs have the labels referenced
by that file.
